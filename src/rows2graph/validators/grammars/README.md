# Vendored grammars

ANTLR4 grammars backing the deployment-free, grammar-based syntax validators
(`rows2graph.validators.{cypher,gremlin}.syntax`). Each is the graph engine's
**own** grammar, so a generated query is validated in-process against the same
structure the engine itself accepts, with no database required. The Python
parsers generated from these files are committed under `../_grammar/generated/`
and are regenerated by `scripts/generate_parsers.sh`.

There is intentionally **no AQL grammar**: ArangoDB publishes no reusable
offline grammar (its parser is hand-written C++/Bison, with semantic actions
entangled with the server runtime), so AQL has no syntax validator. Validate
AQL with `server` / `managed` mode instead (`db.aql.validate` against a live or
auto-provisioned ArangoDB).

## Cypher: `Cypher25Lexer.g4`, `Cypher25Parser.g4`

- Source: Neo4j `cypher-language-support`,
  `packages/language-support/src/antlr-grammar/Cypher25{Lexer,Parser}.g4`
  <https://github.com/neo4j/cypher-language-support>
- License: Apache-2.0 (© Neo4j Sweden AB).
- Start rule: `statements` (anchors `EOF`).
- Edits: none; vendored verbatim, generated as-is for the Python3 target.

## Gremlin: `Gremlin.g4`

- Source: Apache TinkerPop `gremlin-language`,
  `gremlin-language/src/main/antlr4/Gremlin.g4`
  <https://github.com/apache/tinkerpop>
- License: Apache-2.0 (© Apache Software Foundation).
- Start rule: `queryList` (anchors `EOF`).
- Edits: none; vendored verbatim, generated as-is for the Python3 target.

## Regeneration

Dev-time only (needs Java + the ANTLR 4.13.x complete jar):

```bash
scripts/generate_parsers.sh
```

The ANTLR tool version must match the `antlr4-python3-runtime` pin in
`pyproject.toml` (4.13.x). End users and CI need neither Java nor ANTLR, only
the committed parsers and the pure-Python runtime.
