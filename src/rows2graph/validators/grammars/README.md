# Vendored grammars

ANTLR4 grammars backing the deployment-free, grammar-based syntax validators
(`rows2graph.validators.{cypher,gremlin,aql}.syntax`). A generated query is
validated in-process against the language's grammar with no database required.
The Python parsers generated from these files are committed under
`../_grammar/generated/` and are regenerated by `scripts/generate_parsers.sh`.

Cypher and Gremlin use each graph engine's **own** published grammar. AQL is
different: ArangoDB publishes no reusable offline grammar (its parser is
hand-written C++/Flex+Bison, with semantic actions entangled with the server
runtime), so `AQLLexer.g4` / `AQLParser.g4` here are a **hand-port** of that
Flex+Bison parser. The port reproduces the grammar structure for recognition
only and is therefore *best-effort*: it may diverge slightly from ArangoDB's
real parser. The `server` / `managed` AQL validator (`db.aql.validate`) remains
authoritative; the offline check is a fast first pass.

## Cypher: `Cypher25Lexer.g4`, `Cypher25Parser.g4`

- Source: Neo4j `cypher-language-support`,
  `packages/language-support/src/antlr-grammar/Cypher25{Lexer,Parser}.g4`
  <https://github.com/neo4j/cypher-language-support>
- License: Apache-2.0 (© Neo4j Sweden AB).
- Start rule: `statements` (anchors `EOF`).
- Edits: none; vendored verbatim, generated as-is for the Python3 target.

## Gremlin: `Gremlin.g4`

- Source: Apache TinkerPop `gremlin-language`,
  `gremlin-language/src/main/antlr4/Gremlin.g4`
  <https://github.com/apache/tinkerpop>
- License: Apache-2.0 (© Apache Software Foundation).
- Start rule: `queryList` (anchors `EOF`).
- Edits: none; vendored verbatim, generated as-is for the Python3 target.

## AQL: `AQLLexer.g4`, `AQLParser.g4`

- Source: hand-port of ArangoDB's own Flex lexer + Bison grammar,
  `arangod/Aql/tokens.ll` and `arangod/Aql/grammar.y`, pinned to the `3.11`
  branch (the version the managed validator provisions, `arangodb:3.11`)
  <https://github.com/arangodb/arangodb/blob/3.11/arangod/Aql/grammar.y>
- License: Apache-2.0 (© ArangoDB GmbH).
- Start rule: `queryStart` (anchors `EOF`).
- Edits: the Bison precedence ladder is folded into one directly left-recursive
  `expression` rule; `NOT IN` is reassembled in the parser; all semantic actions
  are dropped. ANTLR has no `%nonassoc`, so a few non-associative chains are
  over-accepted (safe for a syntax checker). Not the engine's own grammar file.

## Regeneration

Dev-time only (needs Java + the ANTLR 4.13.x complete jar):

```bash
scripts/generate_parsers.sh
```

The ANTLR tool version must match the `antlr4-python3-runtime` pin in
`pyproject.toml` (4.13.x). End users and CI need neither Java nor ANTLR, only
the committed parsers and the pure-Python runtime.
